# CLAUDE.md

## Project Rules

Always write code for this project using:

* Vue 3
* Vite
* JavaScript (unless TypeScript is specifically requested)
* Composition API
* `<script setup>`
* SCSS for styling
* Node.js + npm

---

## Code Style

Follow these rules:

* Keep code clean, simple, and beginner-friendly
* Prefer easy-to-maintain solutions over complex architecture
* Write code as a junior developer can understand and continue
* Use clear file structure and readable naming
* Avoid unnecessary abstraction and overengineering
* Use reusable components when it makes sense
* Keep components small and focused
* Prefer practical solutions over “perfect” enterprise patterns

---

## Vue Rules

Always use:

* Vue 3 Composition API
* `<script setup>`
* Props and emits in a clean simple way
* Computed and watch only when necessary
* Pinia only if state management is truly needed
* Vue Router for page navigation
* Simple API handling with fetch or axios

Avoid:

* Options API
* overly complex state management
* unnecessary plugins
* unnecessary libraries

---

## Styling Rules

Always use:

* SCSS
* clean spacing and layout
* responsive design
* mobile-first approach
* modern UI/UX principles
* simple professional design
* accessible and readable interfaces

Prefer:

* Flexbox and Grid
* reusable utility classes when helpful
* consistent spacing system
* soft shadows, rounded corners, clean typography
* modern but realistic UI for real projects

Avoid:

* messy inline styles
* random spacing values
* inconsistent layouts
* overly flashy design
* complicated animation unless requested

---

## Development Rules

Always:

* explain terminal commands when needed
* use npm commands compatible with Vite
* write production-ready but simple code
* prefer stable and common packages
* suggest best practices for junior-friendly development
* help debug terminal and Vue errors clearly

When giving terminal commands, assume:

* VS Code is used
* npm is used
* Vite project structure
* Node.js environment

---

## Response Style

Prioritize:

1. Working code
2. Clean structure
3. Easy understanding
4. Good design
5. Best practical solution

Explain only when necessary.

Do not overcomplicate simple tasks.

Prefer code that works well in real projects and is easy to maintain.
